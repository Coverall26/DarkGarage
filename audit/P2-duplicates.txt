  - ee/features/billing/ components -> /api/teams/${teamId}/billing/*

Since App Router takes priority, all SWR hooks and fetch calls already
hit the App Router version. Pages Router versions are dead code.

Verdict: DELETE all 11 Pages Router billing files.

IMPORTANT NOTE: The dataroom and document sub-routes (e.g.,

and must NOT be deleted. Only the base routes listed above are

Pages Router base routes are larger (268 vs 222 for datarooms, 385 vs

(RBAC, rate limiting, structured errors). The extra lines in Pages
Router are likely from older auth patterns.

Verdict: DELETE all 4 Pages Router base route files.
  KEEP: All Pages Router dataroom/document SUB-routes (folders, views,

CATEGORY 5: WEBHOOK HANDLERS (6 routes)
-----------------------------------------
All 6 webhook handlers have nearly identical implementations in both
routers. Both versions have HMAC signature verification. The App Router
versions are slightly larger (more complete error handling with
reportError, structured logging).

CRITICAL: Webhooks are called by EXTERNAL services (Stripe, Resend,
Persona, Plaid, Rollbar, custom). These services are configured to
call a specific URL. Since Next.js routes App Router over Pages Router,
the App Router version is what actually handles requests.

Webhook endpoints and their external callers:
  - /api/webhooks/esign -> Internal e-signature system
  - /api/webhooks/persona -> Persona KYC service
  - /api/webhooks/plaid -> Plaid banking service
  - /api/webhooks/rollbar -> Rollbar error monitoring
  - /api/webhooks/services/[...path] -> Custom incoming webhooks
  - /api/webhooks/signature -> E-signature completion events

NOTE: /api/webhooks/stripe-crm and /api/webhooks/resend exist ONLY
in App Router — no duplicates for those.

Verdict: DELETE all 6 Pages Router webhook files.
